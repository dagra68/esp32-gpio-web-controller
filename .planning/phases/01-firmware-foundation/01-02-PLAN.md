  <how-to-verify>
**Verification A — REST API (FIRM-02):**

Run each curl command and confirm the expected response:

```bash
# A1: Device responds at root
curl http://{DEVICE_IP}/
# Expected: HTML page or JSON — any non-curl-error response confirms web_server is up

# A2: Switch state — get current state of GPIO 3
curl http://{DEVICE_IP}/switch/GPIO%203
# Expected: JSON like {"id":"switch-gpio_3","name":"GPIO 3","state":"OFF","value":false}

# A3: Turn GPIO 3 ON
curl -X POST http://{DEVICE_IP}/switch/GPIO%203/turn_on
# Expected: HTTP 200; check with: curl -v -X POST http://{DEVICE_IP}/switch/GPIO%203/turn_on

# A4: Confirm GPIO 3 is now ON
curl http://{DEVICE_IP}/switch/GPIO%203
# Expected: "state":"ON","value":true

# A5: Turn GPIO 3 OFF
curl -X POST http://{DEVICE_IP}/switch/GPIO%203/turn_off
# Expected: HTTP 200

# A6: Toggle (confirm toggle endpoint works)
curl -X POST http://{DEVICE_IP}/switch/GPIO%203/toggle
# Expected: HTTP 200; state flips each time
```

**Verification B — SSE stream (FIRM-02):**

```bash
# Stream SSE events — let it run for 10 seconds, then Ctrl+C
curl -N -H "Accept: text/event-stream" http://{DEVICE_IP}/events
```

Expected output: A continuous stream of events like:
```
event: ping
data: {}

event: state
data: {"name_id":"switch/GPIO 3","id":"switch-gpio_3","state":"OFF","value":false}

event: state
data: {"name_id":"binary_sensor/GPIO 20","id":"binary_sensor-gpio_20","state":"OFF","value":false}
```

Confirm:
- Events have `name_id` field in format `domain/entity_name` (e.g., `switch/GPIO 3`)
- Events have `state` field with value "ON" or "OFF"
- All 11 entities appear in the initial state dump (may take a few seconds)

**Verification C — SSE via Browser DevTools (FIRM-02 — visual confirm):**

1. Open `http://{DEVICE_IP}` in a browser
2. Open DevTools (F12) → Network tab
3. Filter by "Other" or type `/events` in the filter bar
4. Find the `/events` request — it should show type `eventsource`
5. Click it → EventStream tab
6. Confirm state events appear with `name_id` and `state` fields

Windows (PowerShell):

Test-NetConnection -ComputerName {DEVICE_IP} -Port 6053

Expected: `Connection to {DEVICE_IP} 6053 port [tcp/*] succeeded!`

Note: HA discovery is optional for Phase 1 gate — port 6053 responding (Verification D) is sufficient to confirm native_api is active.
