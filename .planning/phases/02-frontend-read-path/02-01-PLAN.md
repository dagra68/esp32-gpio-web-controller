When opened via a local HTTP server pointed at the esp32-gpio-controller directory, index.html must show a live pin grid of all 11 GPIO entities with real-time state from the device at 10.1.1.162, a connection status banner that reflects the actual SSE connection state, and correct HIGH/LOW and OUTPUT/INPUT labels on every card.

## Context

Phase 1 delivered a live ESPHome device at 10.1.1.162. All API facts are confirmed from live hardware:
- SSE at `/events` sends a `ping` then all 11 `state` events on connect, then pings every 30s
- SSE `state` event JSON: `{name_id, domain, name, value, state}` — use `name_id` as key, `value` (bool) for HIGH/LOW
- CORS is present (`Access-Control-Allow-Origin: *`) — cross-origin fetch from localhost works
- 9 switches (GPIO 0,1,3,4,5,6,7,10,11) + 2 binary_sensors (GPIO 20,21)
- EventSource `readyState`: CONNECTING=0, OPEN=1, CLOSED=2 — CLOSED requires manual reconnect

Critical constraint: `file://` origin may block CORS in some browsers. Always test via `python -m http.server 8080`, not by opening index.html directly.

### Task 2: Create gpio-ui.js

Create `esp32-gpio-controller/gpio-ui.js`.

The file must implement all five requirements (READ-01 through READ-05) as a single self-contained JS module. No `import`/`export`. No external dependencies. Strict execution order: module-level constants, then function declarations, then `document.addEventListener('DOMContentLoaded', connect)`.

**Module-level constants:**

```javascript
const DEVICE_HOST = (location.hostname === 'localhost' || location.hostname === '')
  ? '10.1.1.162'
  : location.host;
const API_BASE = `http://${DEVICE_HOST}`;
```

**State:**

```javascript
const pins = new Map(); // name_id -> {name, domain, state, value}
let initialBurstDone = false;
let es = null;
```

**`setStatus(status)` function:**
- Accepts `'connected'`, `'reconnecting'`, or `'disconnected'`
- Gets `document.getElementById('status-banner')`
- Sets `banner.className = 'status-banner ' + status`
- Sets `banner.textContent`:
  - connected: `''` (empty — banner has `display:none` via CSS class)
  - reconnecting: `'Reconnecting...'`
  - disconnected: `'Disconnected — retrying...'`

**`buildCard(nameId, pin)` function:**
- Creates a `<div class="pin-card">` with `dataset.nameId = nameId`
- Sets `innerHTML` to three child divs:
  - `<div class="pin-name">${pin.name}</div>`
  - `<div class="pin-type ${pin.domain}">${pin.domain === 'switch' ? 'OUTPUT' : 'INPUT'}</div>`
  - `<div class="pin-state ${pin.value ? 'high' : 'low'}">${pin.value ? 'HIGH' : 'LOW'}</div>`
- Returns the card element

**`renderGrid()` function:**
- Gets `document.getElementById('pin-grid')`
- Clears `innerHTML = ''`
- Iterates `pins` Map, appends `buildCard(nameId, pin)` for each entry

**`upsertPin(nameId, pin)` function:**
- Calls `pins.set(nameId, pin)`
- If `!initialBurstDone`: returns early (buffering during initial burst)
- Otherwise: finds existing card with `document.querySelector('[data-name-id="' + nameId + '"]')`
  - If found: calls `existing.replaceWith(buildCard(nameId, pin))`
  - If not found: calls `renderGrid()` as fallback

**`connect()` function:**
- If `es` is not null: call `es.close()` then set `es = null`
- Set `initialBurstDone = false`
- Call `setStatus('reconnecting')`
- Set `es = new EventSource(API_BASE + '/events')`
- Add event listener for `'state'`:
  - Parse `JSON.parse(e.data)` into `d`
  - Call `upsertPin(d.name_id, { name: d.name, domain: d.domain, state: d.state, value: d.value })`
  - Also: if not initialBurstDone, set a 500ms timeout fallback — if ping hasn't fired within 500ms of the first state event, set `initialBurstDone = true` and call `renderGrid()`
- Add event listener for `'ping'`:
  - If `!initialBurstDone`: set `initialBurstDone = true`, call `renderGrid()`
- Set `es.onopen`:
  - Call `setStatus('connected')`
- Set `es.onerror`:
  - If `es.readyState === EventSource.CLOSED`:
    - Call `setStatus('disconnected')`
    - Call `setTimeout(connect, 3000)`
  - Else (CONNECTING — browser is auto-reconnecting):
    - Call `setStatus('reconnecting')`

**Bottom of file:**
```javascript
document.addEventListener('DOMContentLoaded', connect);
```

### Task 3: Smoke test via local server

This task is user-executed. The agent documents the verification procedure and the expected outcomes.

Verification procedure:
1. In the `esp32-gpio-controller/` directory, run: `python -m http.server 8080`
2. Open `http://localhost:8080/index.html` in a browser
3. Open browser DevTools (F12) > Console and Network tabs

Expected outcomes:
- Network tab shows `GET http://10.1.1.162/events` with status 200 and type `eventsource`
- Console shows no uncaught errors
- Grid renders 11 pin cards within ~1 second of page load
- Cards for GPIO 0-11 show "OUTPUT" label in blue; GPIO 20-21 show "INPUT" label in green
- Card for GPIO 3 (was toggled ON in Phase 1) shows "HIGH" in green; others show "LOW"
- No status banner visible (connection is live)
- Kill test: disconnect WiFi on laptop — banner shows "Reconnecting..." within ~5 seconds
- Reconnect test: reconnect WiFi — banner disappears; grid refreshes with current device state

This task requires the device at 10.1.1.162 to be powered and on the network. If the device IP has changed (DHCP renewal), update the hardcoded IP in gpio-ui.js before testing.
